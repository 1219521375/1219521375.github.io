使用GitAction更优雅地写博客

前言

两年前就有搞过自己的Hexo博客，然而后来因为感觉写起来太复杂烦乱，要用一堆命令就弃了（其实就是懒+菜）。

现在终于找到了比较优雅的写博客方式，将博客的源码存放到github仓库，然后每次写博客只需用git push的方式提交到该源码仓库中，然后用GitActions自动部署到博客仓库即可，可以很方便地在不同的电脑上撰写更新博客。

一、提交博客源码到github仓库

根据之前的使用Hexo-github搭建个人博客，我们现在已经有了一个存放博客静态文件的github.io仓库，我们还需要建立一个存放博客源码的仓库，因为源码涉及私人信息，建议设置为私有仓库（github现在已经支持免费的私有仓库搭建）。

即将此文件夹上传至私有源码仓库。可以看到Hexo在创建时已经建立了gitignore文件，去除了node_modules public等非源码文件，我们直接进行上传即可。

二、准备ssh密钥

windows平台下可在C:\Users\用户名\.ssh下找到，.pub后缀的是公钥，无后缀的是私钥。

配置公钥

只要你能正常git push就不用配置了，一般都是配置好的，没有配置可以在github个人设置里配置ssh公钥。

配置私钥

在私有仓库的Settings中的Secrets项，新增一个私钥，将id_rsa中的所有内容复制到value中，命名为HEXO_DEPLOY_PRIVATE_KEY（也可以自己命名，需要和后面workflow中一致）。

三、添加GitAction

在私有源码项目中点击Action栏，添加一个workflow。

github会自动生成一个main.yml文件，将以下代码复制到其中。

点击提交，之后每次push源码后，GitAction就会自动直径这些操作。

此后，只需对私有源码仓库进行git push即可自动更新博客。

更换电脑、重装系统后只需git clone源码项目后使用git add手动添加md文件或者在线添加md文件到/source目录下即可。若需要使用hexo new "blogname"自动生成新博客，需要使用npm安装hexo之后使用命令npm install hexo-deployer-git。
