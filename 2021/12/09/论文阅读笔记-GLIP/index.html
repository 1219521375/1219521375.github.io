论文阅读笔记-GLIP

Abstract

  • 提出了一种用于学习对象级、语言感知和语义丰富的视觉表征的 基础/定位 语言图像预训练(GLIP)模型，基于DyHead。

  • GLIP将目标检测和短语定位统一起来进行预训练，因为目标检测可以被看作是上下文无关的短语基础，而短语基础可以被看作是上下文中的目标检测任务。这种统一带来两个好处：

    • 1)GLIP可以从检测和定位数据中学习，改进任务和引导良好的检测定位模型；
    • 2)GLIP可以利用大量的图文对，以自训练的方式生成定位框，使学习到的表示具有丰富的语义。

  • 在27M个真实数据上对GLIP进行了预训练，这些数据包括3M人工标注和24M网络爬取的图文对。

  • 学习到的特征表示对各种对象级别的识别任务表现出很强的 zero-shot 和 few-shot 迁移能力。

    • 1)直接在COCO和LVIS上评估(训练前没有看到COCO中的任何图像)，GLIP分别达到49.8AP和26.9AP，超过了许多监督基线。
    • 2)微调COCO后，GLIP在VAL上达到60.8AP，在test-dev上达到61.5AP，超过了之前的SOTA。
    • 3)当转移到13个下游目标检测任务时，1-shot GLIP可以与具有全监督的Dynamic Head相媲美。

主要贡献

  • 将检测与定位统一起来，将目标检测重新表述为短语定位。

    如下图，统一后改变了检测模型的输入，使其不仅接收image，还接受一个text prompt，这个prompt描述了检测任务中所有的候选类别，且这些类别很多可能不会出现在图像中。例如下图，在COCO任务中使用的text prompt为一个80词的string，用点 . 进行分割。且使用word-region alignment scores来替换边界框分类，与clip不同的是这里encoder使用了fusion。

  • 通过海量图文数据放大视觉概念。通过一个好的teacher model来对爬取的海量图文对自动生成grounding boxes来扩充预训练数据，其中名词短语用一个NLP parser检测。以此方式可以在27M数据上进行预训练，其中3M是human-annotated fine-grained data，24M是web-crawled image-text pairs。在24M爬取数据中，有78.1M高置信度的phrase-box伪标注，58.4M的不同的名词短语

  • 使用GLIP进行迁移学习：一种模型适用于所有。在COCO2017和LVIS val上zero shot达到49.8和26.9AP. could tune only the task-specific prompt embedding, whilekeeping the model parameters unchanged.

GLIP 具体方法

Unified Formulation统一方式

原有的目标检测方法无论two-stage或是one-stage，可以将loss归纳为

L=Lcls+Lloc,  (1)\mathcal L = \mathcal L_{cls}+\mathcal L_{loc},\ \ (1) L=Lcls​+Lloc​,  (1)

rpn部分loss被归纳进loc中，而其中cls部分loss由box classifier产生，其常用简单linear层表示，可视为：

O=EncI(Img),Scls=OWT,Lcls=loss(Scls;T),  (2)O∈RN×d, W∈Rc×d, Scls∈RN×c, T∈{0,1}N×cO=Enc_I(Img), S_{cls}=OW^T,\mathcal L_{cls}=loss(S_{cls};T),\ \ (2)\\ O\in \mathbb R^{N\times d},\ W\in \mathbb R^{c\times d},\ S_{cls}\in \mathbb R^{N\times c},\ T\in \{0,1\}^{N\times c} O=EncI​(Img),Scls​=OWT,Lcls​=loss(Scls​;T),  (2)O∈RN×d, W∈Rc×d, Scls​∈RN×c, T∈{0,1}N×c

Object detectron as phrase grounding 将目标检测转化为短语定位

将每个检测框与与text prompt中的c个短语对齐。一种简单的prompt是 Prompt = “Detect: person, bicycle, car, … , toothbrush” ，其中每个短语都是潜在类别名。可以根据更好的描述或语言模型的偏好来设计更好的prompt，例如使用BERT初始化language Encoder时， the prompt “person.bicycle. car. … . toothbrush”会表现地更好。

在GLIP模型中，image region和words的匹配分数 SgroundS_{ground}Sground​ 可以表示为

O=EncI(Img),P=EncL(Prompt),Sground=OPT,  (3)O=Enc_I(Img), P=Enc_L(Prompt),S_{ground}=OP^T,\ \ (3) O=EncI​(Img),P=EncL​(Prompt),Sground​=OPT,  (3)

其中 P∈RM×dP\in \mathbb R^{M\times d}P∈RM×d 是language encoder提取的上下文相关的word/token feature，作用类似于公式(2)中的权重矩阵W，故公式3的做法可以看作图1中的右边部分。由image encoder EncIEnc_IEncI​ 和language encoder EncLEnc_LEncL​ 组成的GLIP model使用端到端的方式训练，并用公式1和公式2中的loss进行最小化。

在使用公式2的loss的过程中，需要将 Scls∈RN×cS_{cls}\in \mathbb R^{N\times c}Scls​∈RN×c 替换为 Sground∈RN×MS_{ground}\in \mathbb R^{N\times M}Sground​∈RN×M，并与target T∈{0,1}N×cT \in \{0,1\}^{N\times c}T∈{0,1}N×c 作loss匹配。word token的数量M始终大于text prompt中短语的数量c ，原因有以下四点：

  1. 部分短语phrases 包含多个words, e.g., “traffic light”
  2. 部分单个word的短语会被分割为多个子word token，e.g., “toothbrush” to “tooth#” and “#brush”
  3. 部分是手动添加的token， 如“Detect:”, “,”,等一些language model中的特殊token
  4. a [NoObj] token 会被添加到 tokenized sequence 的末尾

当loss函数是一个二元sigmoid loss的时候，通过使一个正匹配短语的所有子word token都是正匹配，且所有手动添加的标记对所有图像特征都是负匹配，则对所有图像特征进行正匹配。将原本的T拓展为 T′∈{0,1}N×MT'\in \{0,1\}^{N\times M}T′∈{0,1}N×M ，通过使所有子词都是正匹配，如果一个短语是正匹配，并且所有添加的标记对所有图像特征都是负匹配，则对所有图像特征进行正匹配。通过这种方式使得 loss(Sground;T′)loss(S_{ground};T')loss(Sground​;T′) 保持一致。推理中将token 概率平均后作为短语概率。（子word？）

当loss函数是一个多类别cross-entropy loss的时候，根据MDETR中的方法，所有没有正匹配的box proposals被 [NoObj] token匹配。loss(S,T′)loss(S,T')loss(S,T′) 变成一个多标签多类别的CE loss，这种情况下推理中将token 概率相加作为短语概率。

Equivalence between detection and grounding 两者等价性

使用上述方法，可以将任何检测模型转化为短语定位模型，两种视角理论上在训练和推理的时候都是等价的（当潜在类别可以适应到一个prompt的时候，对于某些具体task可以将类别分割到多个prompts）。这一点已经得到验证：the SoTA DyHead detector with Swin-Tiny backbonegives在使用我们的转换方式前后得到相同的性能。所以，由于语言编码器的自由输入，预先训练的短语接地模型可以直接应用于任何目标检测任务，这使得GLIP模型能够以zero-shot的方式转换到任意的检测任务中。

Language-Aware Deep Fusion语言感知的深度融合

公式3中的encoder是相互分开的，只在后期一起计算匹配分数时，作者称这种模型为 late-fusion 模型。而如今的VL架构中视觉语言的融合是十分重要的，所以在图一中部在后几层encoder引入了image和language的深度融合。具体而言，当使用DyHead作为作为image encoder，BERT作为text encoder时，这个深度融合的encoder可以表示为：

\textcolor[rgb]{1,0,0}{O_{t2i}^i,P_{i2t}^i} = X-MHA(O^i,P^i),\ \ i\in \{0,1,..,L-1\}\ \ (4)\\ O^{i+1}=DyHeadModule(O^i\textcolor[rgb]{1,0,0}{+O_{t2i}^i}),\ \ O=O^L,\ \ (5)\\ P^{i+1}=BERTLayer(P^i\textcolor[rgb]{1,0,0}{+P^i_{i2t}}),\ \ P=P^L,\ \ (6)

L是DyHead中DyHeadModule的数量，BERTLayer是在预先训练的BERT之上新添加的BERT层。 O0O^0O0 是vision Backbone输出的视觉特征， P0P^0P0 是language backbone（BERT）生成的token特征。跨模态交流功能由X-MHA模块实现（公式4），然后再与单模态融合，即再公式5，6中的更新方法。在不添加上下文向量（\textcolor[rgb]{1,0,0}{O_{t2i}^i,P_{i2t}^i}作为视觉模态和文本模态）的情况下，模型就退化为 late-fusion 模型。

跨模态注意力模块X-MHA中，每个head通过关注另一个模态来计算模态的上下文向量：

O(q)=OW(q,I),P(q)=PW(q,L),Attn=O(q)(P(q))T/d ,P(v)=PW(v,L),Ot2i=SoftMax(Attn)P(v)W(out,I),O(v)=OW(v,L),Pi2t=SoftMax(AttnT)o(v)W(out,L),O^{(q)}=OW^{(q,I)},P^{(q)}=PW^{(q,L)},Attn=O^{(q)}(P^{(q)})^T/\sqrt d\ ,\\ P^{(v)}=PW^{(v,L)},O_{t2i}=SoftMax(Attn)P^{(v)}W^{(out,I)},\\ O^{(v)}=OW^{(v,L)},P_{i2t}=SoftMax(Attn^T)o^{(v)}W^{(out,L)}, O(q)=OW(q,I),P(q)=PW(q,L),Attn=O(q)(P(q))T/d​ ,P(v)=PW(v,L),Ot2i​=SoftMax(Attn)P(v)W(out,I),O(v)=OW(v,L),Pi2t​=SoftMax(AttnT)o(v)W(out,L),

其中 {W(symbol,I),W(symbol,L):symbol∈{q,v,out}\{W^{(symbol,I)},W^{(symbol,L)}:symbol\in \{q,v,out\}{W(symbol,I),W(symbol,L):symbol∈{q,v,out} 是可学习的参数，分别作用与query，value，输出线性层，起类似的作用。

深度融合编码器(4)-(6)带来两个好处。1)改进了短语定位性能。2)使学习到的视觉特征具有语言感知能力，模型的预测以文本提示为条件。这对于实现让一个model服务于所有下游检测任务的目标至关重要。

Pre-training with Scalable Semantic-Rich Data 利用可扩展的富含语义的数据进行预训练

扩大检测词汇量：gold grounding 数据集包含一个大的视觉概念库，而最大的检测数据集词汇量约2000类。这个大的词汇数据库包含了几乎所有grounded caption中的视觉概念，如Flickr30K包含44518个unique 短语，VGCaption包含110689个unique短语。与额外的2M检测数据相比，0.8M黄金接地数据在检测稀有类别方面带来了更大的改进。

GLIP选择放大grounding 数据而不是检测数据。首先用 gold(human-annotated) detection and grounding data 预训练一个teacher GLIP，然后用这个teacher model 来对网络爬取的image-text data做box预测，并用[NLP parser](Steven Bird, Ewan Klein and Edward Loper: Natural Language Processing with Python, Analyzing Text with the Natural Language Toolkit)来检测名词短语。最后用gold数据和生成的伪grounding数据训练一个student model。

对于为何学生模型表现得比教师模型更好，文章假定在grounding时，教师模型能够利用语言上下文和语言概括能力来准确定位它并不天生知道的视觉概念。一些并未出现在gold数据集中的概念可以通过丰富的语言上下文例如句法结构来提供有根据的猜测，由这些受过教育的猜测来帮助学生模型学习概念。
